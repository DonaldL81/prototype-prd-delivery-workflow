# Codex 项目入口规则

本项目采用固定的产品原型开发流程。任何开发、恢复上下文、阶段切换或长对话后继续工作时，必须先执行以下入口检查。

## 给 AI 的启动提示

当用户要求按本流程开发时，先读取本文件和 `docs/00_workflow.md`，再开始工作。

推荐用户提示词：

```text
请先读取项目根目录 AGENTS.md 和 docs/00_workflow.md，并严格按其中流程执行。
不要直接开发。先根据流程整理项目框架、确认技术栈和开发规范，再输出开发计划。
开发过程中必须持续更新 page-specs.md、decision-log.md 和 PRD.md。
```

## 适用范围

本文件适用于产品原型开发、前端页面开发、PRD 沉淀、可预览交付、可选 Figma 写入等项目。

如果本文件被复制到新项目中，默认仍然生效。新项目必须先完成项目文档初始化，再进入开发。

## 启动检查

1. 先读取 `docs/00_workflow.md` 的“0. 执行版检查卡”。
2. 再读取当前项目的 `docs/project-brief.md`、`docs/page-specs.md`、`docs/decision-log.md`、`docs/PRD.md`，如果文件存在。
3. 回复或继续工作前，先确认当前阶段、下一步和待确认事项。
4. 如果无法判断当前阶段，必须先整理项目状态，不得直接开发。

## 新项目初始化

如果是新项目，或以下文件不存在，必须先创建基础文档骨架：

1. `docs/project-brief.md`
2. `docs/page-specs.md`
3. `docs/decision-log.md`
4. `docs/PRD.md`

初始化后先输出：

```text
当前阶段：
已创建文档：
下一步：
待用户提供：
```

用户提供需求概括、需求背景、业务背景和系统框架后，才进入项目框架整理。

## 开发门禁

进入页面开发前必须满足：

1. 项目框架已确认。
2. 技术栈和开发规范已确认。
3. 开发计划已确认。
4. 当前页面已询问参考资料。
5. 当前页面已输出页面开发确认卡。
6. 用户已确认页面字段、交互、接口、后端逻辑、数据来源和验收标准。

## 沉淀门禁

每个页面完成后必须同步：

1. 更新 `docs/page-specs.md`。
2. 更新 `docs/decision-log.md`。
3. 将正式逻辑增量写入 `docs/PRD.md`。
4. 区分正式开发逻辑和原型演示逻辑。

## 失忆恢复规则

如果上下文压缩、对话过长、阶段不清楚或无法确定下一步：

1. 停止继续开发。
2. 重新读取 `docs/00_workflow.md` 的执行版检查卡。
3. 读取已有项目文档恢复状态。
4. 先输出“当前阶段 / 已完成 / 下一步 / 待确认”，再继续。

## 最低交付要求

除非用户明确调整，项目完成时至少交付：

1. 可预览页面。
2. `dist/` 构建产物。
3. 单文件 HTML。
4. `docs/PRD.md`。
5. `docs/project-brief.md`。
6. `docs/page-specs.md`。
7. `docs/decision-log.md`。

交付前必须询问是否需要写入 Figma。
